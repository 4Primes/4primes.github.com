--- title: "Play Model S\xC4\xB1n\xC4\xB1flar\xC4\xB1 Aras\xC4\xB1ndaki \xC4\xB0li\xC5\x9Fkilerin Bulunmas\xC4\xB1" wordpress_url: http://blog.4primes.com/?p=588 layout: post date: 2011-08-06 14:27:38 +03:00 wordpress_id: 588 --- Bir önceki yazımda play framework model sınıfları arasındaki ilişkilerden bahsedip, bu ilişkilerin bulunması kısmını sonraki yazımda ele alacağımı belirtmiştim. Şimdi bu ilişkileri nasıl bulduğumuz, hangi yapıları kullandığımız ve verileri nasıl tuttuğumuzdan bahsetmeye çalışacağım. Play model sınıfları arasındaki ilişkilerin annotationlar yardımıyla sağlandığını biliyoruz. Eğer modellerin içerisinde bu annotationları arama yoluna gidersek çözüme ulaşacağımızı düşünüp bu yolu izledik. Play modelleri @Entity annotation'ı ile belirtilir, bunu bir önceki yazımda verdiğim kod örneklerinde net bir biçimde gördük. @Entity annotation'ı ile başlayan modellerde arama yapıp bize gerekli olan ilişki türlerini bulmamız play'in bize sağladığı imkanlar doğrultusunda çok da zor olmadı. İsterseniz ilk önce bu verileri nasıl tuttuğumuza bir bakalım, sonrasında ilişki türlerini bulma işlemlerini inceleriz. 3 farklı class yapısı oluşturduk, bunlara Model, Member ve Association isimlerini verdik. Model class'ı @Entity annotation'ından sonra gelen modellerin adlarını ve tablo adları gibi diğer bilgilerini tuttuğumuz classtı. Member class'ı ise bu her bir modelin memberları hakkında bilgi veren, ve eğer varsa diğer modellerin memberları ya da direk modellerle olan ilişkileri tutacağımız Associaation class'ına erişeceğimiz association bilgisi içeren classtı. İlişkileri tuttuğumuz Association class'ı da hedef modelin adı ve ilişki tipi bilgisini içermektedir. Şimdi bu classlara gör göz atalım. [sourcecode lang="java"] public class Model { String name; String tableName; } public class Member { String name; String columnName; String type; Association association; } public class Association { String targetModelName; String type; }[/sourcecode] Şimdi de bu classlar içindeki memberları setlenecek verileri yani ilişkilerin bulunup bu planladığımız veri formatında tutulması işlemine bir göz atalım. Bu yapının bir modül olacağından söz etmiştim, ama ilk etapta bir plugin yapısı oluşturduk ve o yapıyı kullandık. Modüle dönüştürme kısmını daha sonra yapacağız. Evet bir plugin yapısı oluşturduk, bu pluginin uygulama başlarken çalışması ve taramaları yapması için; [sourcecode lang="java"] public class MyPlugin extends PlayPlugin { @Override public void onApplicationStart() { ... }}[/sourcecode] dememiz yeterli oldu. @Override ve onApplicationStart fonksiyonu bu imkanı bize sağlıyor. Peki uygulama çalışmaya başlar başlamaz bizim pluginimiz ne yapıyor? Play'in ClassLoader isimli bir fonksiyonu var. Classlar içerisinde gezinmek için oluşturulmuş bir fonksiyon, yani tam bize lazım olan fonksiyon. İşte biz bu yapıya, @Entity annotation'ına göre arama yaptırıp her bir modeli ayrı ayrı listeleyerek işleme başladık. [sourcecode lang="java"] List entities = Play.classloader.getAnnotatedClasses(Entity.class);[/sourcecode] Bu classları listeleyip for() yapısı kullanarak hepsinin içerisinde teker teker arama yaptık. Aradığımız annotationlar ise şunlar: @Column, @JoinColumn, @OneToOne, @OneToMany, @ManyToOne ve @ManyToMany. Bu arama işlemlerini @OneToOne ve @OneToMany için şöyle yaptık: [sourcecode lang="java"] if (annotation.annotationType().equals(OneToMany.class)) { OneToMany oneToMany = (OneToMany) annotation; Association association = new Association(); association.type = "OneToMany"; Type genericType = field.getGenericType(); if (genericType instanceof ParameterizedType) { ParameterizedType aType = (ParameterizedType) genericType; Type fieldArgType = aType.getActualTypeArguments()[0]; association.targetModelName = ((Class) fieldArgType).getSimpleName(); } member.association = association; } else if (annotation.annotationType().equals(OneToOne.class)) { OneToOne oneToOne = (OneToOne) annotation; Association association = new Association(); association.type = "OneToOne"; association.targetModelName = field.getType().getSimpleName(); member.association = association; }[/sourcecode] Target model ile ilişkisi Many türünden olan; @OneToMany ve @ManyToMany, yapılarında hedefteki modelin adını çekme kısmında biraz sorun yaşadık. Hazır buraya değinmişken, StackOverflow'dan da bahsediyim. Yaşadığımız sorunu bu siteden araştırıp bulduk. Bir çok programlama dili ve framework alanında gerçekten expert seviyesine gelmiş şahısların bulunduğu bir ortam StackOverflow. Sıkıntınızı araştırıp ya da düzgün bir İngilizce ile sorduğunuzda çözüm bulamamanız neredeyse imkansız. Yukarıdaki şekilde diğer annotationlar için de teker teker tarama yaptıktan sonra elimizde verileri oluşturduğumuz 3 farklı class'ın memberlarına setleyip, ilk etapta bu veri formatında sakladık, daha sonra JSON formatına dönüştürdük. Bu verilerin JSON formatındaki yapısına örnek teşkil etsin diye, User modeli için oluşan yapıya bir göz atalım: [sourcecode lang="javascript"] "User": { "name": "User", "tableName": "User", "superClassName": "CreatedType", "isMappedSuperClass": false, "members": { "events": { "nullable": true, "unique": false, "name": "events", "columnName": "events", "type": "List", "association": { "targetModelName": "Event", "type": "OneToMany" } }, "friends": { "nullable": true, "unique": false, "name": "friends", "columnName": "friends", "type": "List", "association": { "targetModelName": "Friend", "type": "ManyToMany" } }, "fullname": { "nullable": true, "unique": false, "name": "fullname", "columnName": "fullname", "type": "String" }, "username": { "nullable": true, "unique": true, "name": "username", "columnName": "username", "type": "String" } } }[/sourcecode] İyi çalışmalar.
